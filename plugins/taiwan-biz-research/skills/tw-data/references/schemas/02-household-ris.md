- 年度格式：**民國 ROC**（路徑必填，`114`）
- 最新可得：**ROC 114（2025 年底）**，7,752 筆，4 頁。ROC 115 回 `OD-0102-S 查無資料`

回應信封（ROC 114 page 1 實測）：
```json
{"responseCode":"OD-0101-S","responseMessage":"處理完成","totalPage":"4","totalDataSize":"7752","page":"1","pageDataSize":"2000"}
```

| 項目 | 值 |
|---|---|
| 單位 | 戶數欄＝戶；男/女欄＝人 |
| 粒度 | 村里（最細），需自行聚合到區→縣市 |
| 筆數 | ROC 114：7,752；ROC 113：7,747 |
| 分頁 | 每頁 2,000 筆，第 4 頁 1,752 筆 |
| 全國普通住戶（ROC 114 實算） | **9,851,033 戶**（22 縣市） |

**坑**
- **同一端點跨年換欄名**：ROC 113 中文鍵、ROC 114 英文鍵。硬編碼任一組都會在換年時無聲拿到 0。必須兩種拼法都讀。
- **錯誤以 HTTP 200 回傳**，靠 body 的 `responseCode` 判斷（`OD-0101-S` 成功／`OD-0102-S` 查無資料）。看 status code 會誤判。
- **`$top` 被靜默忽略**：`?page=1&$top=5` 仍回 2,000 筆。只能用 `page` 分頁。
- 超出範圍的 page（如 page=5）回 `OD-0102-S`，不是空陣列，需與「該年未發布」區分（兩者 responseCode 相同）。
- 「戶數」定義三分：`共同生活戶` + `單獨生活戶` = **普通住戶**（有家庭預算，市場推估用這個）；`共同事業戶`（宿舍、營區）要排除。
- `site_id` / `區域別` 是「縣市+區」黏在一起（`新北市板橋區`），無獨立縣市欄。前 3 字切縣市可行（所有縣市名皆 3 字），但這是脆弱假設。
- 用「臺」不用「台」。
- **22 縣市，含金門縣、連江縣**——比家庭收支調查多兩縣，join 時會產生無支出資料的孤兒列。
- 統計年 114 = 民國 114 年**底**（2025-12），與家庭收支調查 2024 年**全年**不同期，混用時有一年時差。

### (b) 最新可得年度 — 實際確認結論

**家庭收支調查系列（9420 / 9415 / 6588 / 6565）：最新為 2024 年（民國 113 年）。2025 年（114 年）尚未發布。**

證據：
1. 四支 CSV 於 2026-07-18 實際下載，最末列一律為 `2024`。
2. data.gov.tw 6588 詮釋資料更新時間 **2025-09-10**，對應 113 年結果的發布批次（113 年綜合統計於 2025-08 發布）。更新頻率標示「每 1 年」。
3. 114 年資料預計 2026-08 發布，尚未到。

**RIS 村里戶數 ODRP019：最新為 ROC 114（2025），ROC 115 明確回 `查無資料`。**

**陷阱端點（勿當作年度證據）**
- `https://ws.dgbas.gov.tw/win/fies/a11.asp?year={N}` 對**任意** year 值都回 HTTP 200 且 **byte 數完全相同**（22,685），標題文字直接把參數 render 進去（`114年家庭收支調查結果綜合分析`）。頁面存在 ≠ 該年有資料。
- 該頁上的報告檔連結（`doc/result/{年}/a11/Preface.doc`）**113、114 皆回 404**，整條路徑已失效，不能用來探測年度。
- `https://ws.dgbas.gov.tw/win/fies/214.asp`（常用歷年資料）已停更，最新只到 **2021**，不反映實際最新年度。
- `ws.dgbas.gov.tw` 全站為 **Big5** 編碼（非 UTF-8），且憑證鏈不完整，`WebFetch` 會以 `unable to verify the first certificate` 失敗；須用 curl。

**唯一可靠的年度判定法**：直接抓 CSV 讀最後一列。
